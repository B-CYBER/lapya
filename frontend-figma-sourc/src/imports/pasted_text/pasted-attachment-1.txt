The Lapya web app currently stretches all content to the full browser width.
This makes inputs, cards, and lists feel sparse, lost in white space, and
"AI-generated." Fix this across the entire app using strict max-width
containers and proper responsive scaling.

═══════════════════════════════════════════════════════════════
GLOBAL LAYOUT RULES — apply to EVERY page
═══════════════════════════════════════════════════════════════

CONTAINER SYSTEM:

1. Outer page wrapper: full width, warm white background (#FBFAF7)

2. Inner content container with these strict max-widths by context:
   • Auth screens (sign up, sign in, onboarding):  max-width 440px, centered
   • Forms and input-heavy screens:                max-width 520px, centered
   • Reading/content pages (meal plan, care circle, profile, history):
                                                    max-width 720px, centered
   • Dashboard / multi-column views (dietitian, home with sidebars):
                                                    max-width 1120px, centered
   • Marketing/landing page sections:              max-width 1200px, centered

3. Horizontal padding on the inner container:
   • Mobile (<640px):     16px
   • Tablet (640-1024px): 24px
   • Desktop (>1024px):   32px

4. Vertical section spacing:
   • Between major sections: 64px desktop, 48px tablet, 32px mobile
   • Between cards in a list: 16px on all viewports
